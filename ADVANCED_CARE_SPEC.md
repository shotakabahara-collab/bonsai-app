# BONSAI 部位別手入れ・病害虫・古木技法仕様 v1

## 目的

盆栽全体を一括で「剪定済み」「針金あり」と扱わず、プレイヤーが触れた位置・強度・方向を部位単位で保存し、作品画像・評価・銘木録へ反映する。

## 部位構造

| ID | 表示名 | 種別 |
|---|---|---|
| `apex` | 頂部 | 枝・葉棚 |
| `first_left` | 第一枝（左下） | 枝・葉棚 |
| `second_right` | 第二枝（右中） | 枝・葉棚 |
| `third_left` | 第三枝（左中） | 枝・葉棚 |
| `back_branch` | 背枝 | 枝・葉棚 |
| `front_branch` | 前枝 | 枝・葉棚 |
| `trunk` | 主幹 | 幹 |
| `nebari` | 根張り・根域 | 根域 |

各部位は葉量、健康度、剪定段階、枝角度、針金、病気、害虫、神、傷跡、更新日時を保持する。

## 剪定

- 軽剪定・中剪定・強剪定を部位ごとに指定する。
- 実行前に不可逆であることを確認する。
- 実行後は葉量、部位健康度、全体樹勢、作業ストレス、剪定履歴へ反映する。
- 従来の剪定ボタンも部位選択画面へ接続し、一括剪定は行わない。

## 針金

部位と強度に加え、次の方向を指定する。

- 下げる
- 上げる
- 左へ流す
- 右へ流す
- 手前へ出す
- 奥へ引く

軽針金・強針金・取り外しを持ち、長期の強針金では食い込み跡が残る場合がある。従来の針金ボタンも部位指定へ統合する。

## 病気

初期実装は次の3種。

- 葉枯れ症状
- すす病
- 根腐れ兆候

発生条件には水分、過湿、作業ストレス、葉の過密度、季節を用いる。病気は該当部位の健康度と全体樹勢を低下させ、治療費を支払って回復させる。

## 害虫

初期実装は次の3種。

- アブラムシ
- ハダニ
- カイガラムシ

夏季、乾燥、葉の過密、樹勢低下などで発生率が上がる。発生部位は画像上の局所表示と警告カードへ反映する。

## 神・舎利

### 神

枝を意図的に枯らして白骨化させる。対象枝には中剪定以上、または十分に少ない葉量と樹勢60以上が必要。実行後は葉量・健康度が0となり、針金を外し、元には戻せない。

### 舎利

主幹の左右どちらへ施すかを指定する。一級盆栽師以上、樹勢70以上が必要。最大3段階まで広げられ、元には戻せない。

## 病害虫イベント

- 12時間単位で一度だけ判定する。
- 判定は保存データから決定的に生成し、再読込による引き直しを防ぐ。
- 発生時は生涯年表と部位履歴へ記録する。
- 同じ部位へ病気と害虫を無制限に重複させない。

## 画像反映

現在の実写静止画へ、次の局所表現を重ねる。

- 剪定部位：対象葉棚の密度低下表示
- 針金：対象枝と指定方向に沿った銅線表示
- 病気：局所的な褐色・黒色変化
- 害虫：対象部位の小さな虫害表示
- 神：白骨化した枝
- 舎利：幹に沿った白木部分
- 樹勢・水分：作品写真全体の彩度と明度を控えめに調整

各状態は `visualSignature` として保存し、将来の実写状態画像アセット選択キーに利用する。局所オーバーレイは完成画像群が揃うまでの移行実装であり、最終版では同一個体の状態別実写画像へ順次置換する。

## 保存互換

既存の `bonsai_live_1` を維持する。旧画面の水やり・施肥などが保存を行っても、`advanced` 部位データを失わないよう Storage ブリッジでマージする。

## 実装ファイル

- `advanced-care.js` — 部位データ、操作、イベント、UI、画像オーバーレイ
- `advanced-care-bridge.js` — 旧剪定・針金ボタン統合、保存互換保護
- `index.html` — 上記モジュールの読込
- `sw.js` — オフラインキャッシュ
- `.github/workflows/apply-advanced-care.yml` — 接続・不可逆操作・HTTP検査
- `.github/workflows/pages.yml` — 公開後検査
