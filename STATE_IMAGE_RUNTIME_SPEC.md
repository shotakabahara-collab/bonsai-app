# BONSAI 作品状態画像ランタイム仕様 v1

## 目的

盆栽の現在値を数値だけで表示せず、部位別剪定・針金・病害虫・神・舎利・樹勢・水分を一枚の状態画像へ合成し、作品の変化を銘木録へ残す。

## 基本原則

- 同じ個体写真を基準にし、幹・根張り・主要枝・カメラ位置を維持する。
- 画像全体への単純な色調変更だけで剪定を表現しない。
- 剪定、針金、病変、虫害、神、舎利は対象部位の座標へ局所反映する。
- 不可逆操作後の画像を端末内へ自動保存する。
- 実写状態アセットが存在する場合は将来それを優先し、存在しない状態は合成ランタイムで補完する。

## 対象部位

- 頂部
- 第一枝
- 第二枝
- 第三枝
- 背枝
- 前枝
- 主幹
- 根張り・根域

各部位には、画像上の中心、作用範囲、枝方向の基準パスを持たせる。

## 画像合成

### 剪定

対象葉棚の範囲内で葉量を段階的に減らし、枝の流れを残す。軽剪定、中剪定、強剪定および保存された葉量を参照する。

### 針金

対象枝の基準パスへ銅線を描画する。軽針金・強針金を線幅で区別し、保存された方向・角度と紐付ける。

### 病気

葉枯れは褐色化、すす病は暗色斑、根腐れ兆候は根域と表土の暗色化として局所表示する。

### 害虫

アブラムシ、ハダニ、カイガラムシは対象葉棚へ微細な虫害痕として表示する。大きな絵文字や説明用アイコンは作品画像へ重ねない。

### 神

神にした枝を白骨化した枝として描画する。対象枝の生きた葉量は復元しない。

### 舎利

主幹の指定側へ、段階に応じた幅の白木部分を幹の流れに沿って描画する。

### 樹勢・水分

樹勢は彩度・明度・葉の生命感、水分は表土・苔・全体の乾燥感へ控えめに反映する。

## 状態画像の自動保存

IndexedDB `bonsai_state_images_v1` に最大80件を保存する。

保存対象には次を含む。

- 初期状態
- 部位別剪定
- 部位別整姿
- 病気・害虫の発生または回復
- 神・舎利
- 鉢替え
- 季節・樹勢・水分の段階変化
- 展覧会出展・受賞
- プレイヤーによる手動保存

各記録は画像、保存時刻、理由、木の銘、樹種、鉢、`visualSignature`、状態投影データを保持する。

## 銘木録UI

画面上の「作品画像」から、保存済み状態画像を時系列で閲覧できる。画像は端末内に保存し、サーバーへ自動送信しない。

## 保存互換

既存キー `bonsai_live_1` を維持する。状態画像は別のIndexedDBへ保存し、既存セーブデータの容量・復旧処理へ影響させない。

## 実装ファイル

- `state-image-runtime.js` — 状態画像合成、監視、自動記録、ギャラリー
- `advanced-care.js` — 部位状態と不可逆操作
- `advanced-care-bridge.js` — 既存操作との統合と保存互換
- `sw.js` — オフラインキャッシュ
- `.github/workflows/apply-state-image-runtime.yml` — 接続・構文・HTTP検査

## 現段階の限界

現在のランタイムは実写基準写真へ局所加工を施す方式である。枝構造そのものを撮影・生成した完全な状態別実写アセットではない。最終版では同一個体の高解像度状態画像群を制作し、`visualSignature` による完全一致アセットを優先表示する。

## 次工程

1. 正常な高解像度基準画像をバイナリアセットとして登録する。
2. 部位別マスクを基準画像へ手動校正する。
3. 剪定強度ごとの同一個体状態画像を制作する。
4. 鉢差分、季節差分、病害虫差分を高解像度アセット化する。
5. 大会受賞時に状態画像を受賞記録へ固定保存する。
6. ユーザー間大会へ状態画像スナップショットを送るサーバー仕様を追加する。
